Zero Trust–Aligned Identity Attack Evaluation

An Awareness-Driven Evaluation of Identity-Based Attacks in a Hardened Active Directory Environment

Final Year Project – Cybersecurity

Introduction

Active Directory (AD) remains the backbone of enterprise identity and access management. It provides centralized authentication, authorization, and account control in Windows-based infrastructures.

Identity-based attacks such as credential misuse, LDAP enumeration, and privilege escalation are among the most impactful threats in enterprise environments.

Modern organizations increasingly adopt Zero Trust principles, enforcing least privilege, policy-based hardening, segmentation, and secure-by-default configurations.

Problem Statement

While many academic demonstrations focus on exploiting misconfigured Active Directory environments, modern enterprises deploy hardened configurations aligned with Zero Trust principles. There is limited evaluation of how identity-based attacks behave under enforced security controls.

Objectives

  • Deploy a hardened Active Directory lab environment
  • Simulate identity-based attack attempts
  • Preserve realistic enterprise security settings
  • Evaluate policy enforcement effectiveness
  • Analyze Windows Security log telemetry
  • Map findings to Zero Trust principles

Proposed Methodology

1. Lab Architecture

  • Windows Server Domain Controller
  • Windows 11 Domain Client
  • Kali Linux Attacker System
  • Controlled same-subnet environment

2. Security Hardening

  • GPO-based hardening
  • LDAP signing enforcement
  • Kerberos integrity enforcement
  • NTLM restrictions
  • Windows Firewall enabled

3. Attack Evaluation

  • Low-privilege credential usage
  • Credential-based reconnaissance
  • LDAP interaction attempts
  • Security log monitoring
  • No disabling of protections

Results

Attack Attempt Observed Outcome Enforced Control
LDAP Enumeration Blocked LDAP Signing Enforcement
Kerberos Interaction Integrity Verified Kerberos Integrity Policy
Lateral Access Restricted Firewall Segmentation
Privilege Escalation Not Achieved Least Privilege Enforcement

Zero Trust Alignment

Assume Breach: Attack simulated using valid compromised credentials.

Least Privilege: Standard domain user without elevated permissions.

Explicit Policy Enforcement: GPO-based hardening and authentication integrity.

Micro-Segmentation: Firewall-enforced access restrictions.

Continuous Monitoring: Windows Security log analysis.

Conclusion

The hardened Active Directory environment demonstrated effective enforcement of Zero Trust–aligned controls.

Identity-based attack attempts were significantly restricted under enforced security policies.

The study highlights the importance of evaluating defensive effectiveness and failure behavior in modern enterprise security research.
